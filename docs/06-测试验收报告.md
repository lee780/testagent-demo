## 三、缺陷记录

### 3.1 本轮测试发现并修复的缺陷

| 缺陷编号 | 模块 | 描述 | 严重度 | 状态 |
|---------|------|------|-------|------|
| BUG-NEW-001 | subscribe.ts | `mapPiEventToSSE` 的 `text_delta` 和 `thinking_delta` 分支检查 `(ame as any).delta` 属性，但 `AssistantMessageEvent` 接口定义的是 `text` 和 `thinking` 字段，导致这两个分支永远返回 null | 高 | **已修复** |

**缺陷详情（BUG-NEW-001）**：

- **文件**：`/Users/claude/testagent/TestAgent-PI-main/src/agent-core/subscribe.ts`
- **位置**：第 51 行和第 57 行（`mapPiEventToSSE`），第 125、128 行（`mapPiEventToFrontendSSE`）
- **根因**：代码检查 `(ame as any).delta` 属性，但 pi-mono SDK 的 `AssistantMessageEvent` 对象中，文本内容存放在 `ame.text`（text_delta 类型）和 `ame.thinking`（thinking_delta 类型）字段，`.delta` 属性不存在，导致条件永远为 false，返回 null
- **影响**：AI 流式输出消息（文本和思考过程）无法转发到前端 SSE 流，前端收不到 agent_message / agent_thinking 事件，对话框不显示 AI 响应内容
- **修复**：将判断条件改为 `(ame.text ?? (ame as any).delta)` 和 `(ame.thinking ?? (ame as any).delta)`，取内容时同样优先用 `ame.text` / `ame.thinking`，保留对旧格式的向后兼容
- **验证**：修复后 `npm test` 中 subscribe.test.ts 的 2 个失败用例全部通过，共 97/97 PASS

## 四、修复记录

### 修复 1：subscribe.ts — text_delta / thinking_delta 事件映射错误

**修复文件**：`/Users/claude/testagent/TestAgent-PI-main/src/agent-core/subscribe.ts`

**修复内容**（两处 if 条件 + 内容提取，共 4 处）：

```typescript
// 修复前（mapPiEventToSSE 函数内）
if (ame.type === "text_delta" && (ame as any).delta) {
  return {
    type: "agent_message",
    data: { content: (ame as any).delta, delta: true },
  };
}
if (ame.type === "thinking_delta" && (ame as any).delta) {
  return {
    type: "agent_thinking",
    data: { content: (ame as any).delta, delta: true },
  };
}

// 修复后（优先使用接口定义字段，保留旧格式向后兼容）
if (ame.type === "text_delta" && (ame.text ?? (ame as any).delta)) {
  return {
    type: "agent_message",
    data: { content: ame.text ?? (ame as any).delta, delta: true },
  };
}
if (ame.type === "thinking_delta" && (ame.thinking ?? (ame as any).delta)) {
  return {
    type: "agent_thinking",
    data: { content: ame.thinking ?? (ame as any).delta, delta: true },
  };
}
```

相同修复也应用于 `mapPiEventToFrontendSSE` 函数（第 125、128 行）。

### 5.5 E2E 根因修复记录
