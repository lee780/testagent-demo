## 三项测试结果

### 1. 测试用例生成（BVA 边界值）

提示词：为信用评分接口生成 3 条边界值测试用例，包含测试目标、输入参数、预期结果。

| 指标        | Model A            | Model B        |
| --------- | ------------------ | -------------- |
| 输出格式      | Markdown 表格（更结构化）  | Markdown 标题段落  |
| 推理 tokens | **1293**           | **0**          |
| 总 tokens  | 1552               | 241            |
| token 效率  | 差（推理占 83%）         | 优（无推理开销）       |
| 用例质量      | 三条 BVA，含 JSON 参数示例 | 三条 BVA，含字段语义说明 |

> 两个模型均覆盖最小边界、最大边界、越界三类场景，内容质量持平，但 Model A 消耗 **6.4x tokens**。

***

### 2. 工具调用（Function Call — Agent 核心能力）

提示词：调用 `run_test_suite` 工具，参数 suiteName=信用评分接口, mode=systematic, caseCount=5。

| 指标                | Model A      | Model B      |
| ----------------- | ------------ | ------------ |
| 调用是否正确            | ✅ 参数完整准确     | ✅ 参数完整准确     |
| `finish_reason`   | `tool_calls` | `tool_calls` |
| 推理 tokens         | 120          | **0**        |
| completion tokens | 181          | **48**       |
| token 效率          | —            | **3.8x 更高**  |

> 工具调用是 TestPilot Agent 最高频操作。Model B 每次 tool call 节省 **73% token 消耗**，在多轮 Agent 场景下差距会指数级放大。

***

## 综合评估

| 维度             | Model A       | Model B  | 胜出       |
| -------------- | ------------- | -------- | -------- |
| **工具调用准确性**    | ✅ 准确          | ✅ 准确     | 持平       |
| **测试用例质量**     | 略优（表格格式）      | 良好       | A 微胜     |
| **代码生成质量**     | 完整但冗余         | 简洁可用     | 持平       |
| **Token 效率**   | 低（大量隐式推理）     | 高（无推理浪费） | **B 大胜** |
| **响应延迟**       | 较高（推理增加耗时）    | 较低       | **B 胜**  |
| **成本**         | 高（按 token 计费） | 低 3–14x  | **B 大胜** |
| **Agent 多轮适配** | 较差（每步推理累积）    | 优秀       | **B 大胜** |

***

## 结论

**Model B（qwen3-coder-480b-a35b-instruct）综合更优**，适合 TestPilot Agent 场景：

- 工具调用与 Model A 完全等价（核心能力无退步）
- Token 消耗降低 73%–93%，显著降低 API 成本
- 无隐式推理开销，Agent 多步调用延迟更低

**已于 2026-03-26 完成切换**，当前项目使用模型：`qwen3-coder-480b-a35b-instruct`
