# TestAgent-PI 产品介绍文档

## 一、产品概述

**TestAgent-PI** 是一款基于 pi-mono 原生 AI Agent 框架构建的智能代码分析与接口测试平台。它将大语言模型（LLM）的推理能力与专业的代码分析、API 测试、数据库操作工具深度融合，通过自然语言交互的方式，帮助开发者和测试工程师高效完成代码理解、接口验证、质量分析等工作。

---

## 二、核心能力

### 1. 智能对话与流式响应
- 支持与 AI Agent 进行自然语言对话
- 基于 Server-Sent Events（SSE）实现实时流式输出，响应及时、体验流畅
- 支持思考过程可视化（Thinking 模式），展示 Agent 的推理步骤
- 支持文件上传，Agent 可结合附件内容进行分析

### 2. 代码智能分析
- **符号搜索**：基于全文检索（FTS5）快速定位函数、类、变量等代码符号
- **调用链分析**：追踪方法调用关系，理解代码执行路径
- **注解查找**：按注解类型扫描代码，快速定位配置项和特性声明
- **源码获取**：精准提取方法源码，支持上下文理解

### 3. API 接口测试
- 自动化发起 HTTP 请求，支持断言验证响应内容
- JSON Path 提取与数据比对
- 捕获接口性能指标（响应时间、状态码等）
- 支持复杂测试场景的链式编排

### 4. 数据库操作
- 安全连接数据库，执行 Schema 查询和数据检索
- 内置生产环境保护机制，自动阻断危险连接
- 支持 SQL 执行与结果分析

### 5. 任务编排与多阶段执行
- 支持将复杂任务分解为多阶段子任务树
- 动态任务创建、状态跟踪与进度可视化
- Worker 模式支持并行子任务执行
- CoordinatorPlan 机制管理多轮协作计划

### 6. 输出文件管理
- Agent 执行结果可写入结构化文件，供用户下载
- 工作区隔离，每个用户/会话拥有独立的文件空间
- 自动轮换清理（保留最近 7 天的历史文件）

## 四、主要功能模块

### 4.1 用户认证与权限
- 注册/登录，JWT 令牌鉴权
- 用户角色：普通用户（USER）/ 管理员（ADMIN）
- 账号状态管理：ACTIVE / INACTIVE / BANNED

### 4.2 会话管理
- 多会话支持，每个会话保存完整的消息历史
- 会话标题自动生成（LLM 驱动）
- 支持删除会话及清理关联资源

### 4.3 Agent 执行引擎
- 支持多 LLM 提供商：Anthropic Claude、OpenAI 兼容接口、企业内网模型
- Thinking 级别可配置：off / low / medium / high
- 基于 JSONL 格式的会话持久化，支持断点续跑
- 优雅降级：pi-mono 未安装时自动切换至独立运行模式

### 4.4 实时事件流
- Agent 执行过程中的思考、工具调用、结果均通过 SSE 实时推送
- 支持中断正在运行的 Agent 任务
- AsyncQueue 模式确保事件有序传递

### 4.5 工作区管理
- 每个用户 + 会话拥有独立的工作目录
- 包含：Session 文件、输出文件、上传文件、临时暂存区
- 自动化生命周期管理（创建、清理、轮换）

## 五、API 接口说明

### 认证接口
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/auth/register` | 用户注册 |
| POST | `/api/auth/login` | 用户登录，返回 JWT |
| POST | `/api/auth/logout` | 退出登录 |

### 对话接口
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/chat/send` | 发送消息（非流式） |
| POST | `/api/chat/stream` | 发送消息（SSE 流式） |
| POST | `/api/chat/upload` | 上传文件至会话 |
| DELETE | `/api/chat/interrupt/{replyId}` | 中断 Agent 执行 |

### 会话管理接口
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/conversation` | 获取会话列表 |
| GET | `/api/conversation/{id}` | 获取会话详情 |
| DELETE | `/api/conversation/{id}` | 删除会话 |

### 系统接口
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/health` | 健康检查 |

## 六、快速开始

### 环境要求
- Node.js v18+（推荐 v22+）
- pnpm v9.0+
- PostgreSQL
- Redis

### 本地开发启动

```bash
# 1. 安装依赖
pnpm install

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env，填写数据库、Redis、JWT 等配置

# 3. 初始化数据库
pnpm prisma:generate
pnpm prisma:migrate

# 4. 启动后端
pnpm dev

# 5. 启动前端（另开终端）
cd frontend
npm install
npm run dev
```

### Docker 一键部署

```bash
# 使用 Docker Compose 启动全栈服务
docker-compose up -d
```

服务默认端口：
- 后端 API：`39996`
- 前端页面：`39997`

### 关键环境变量

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| `PORT` | 后端监听端口 | `8000` |
| `DATABASE_URL` | PostgreSQL 连接串 | `postgresql://user:pass@localhost:5432/testagent` |
| `REDIS_URL` | Redis 连接地址 | `redis://localhost:6379` |
| `JWT_SECRET` | JWT 签名密钥 | `your-secret-key` |
| `LLM_PROVIDER` | LLM 提供商 | `anthropic` |
| `LLM_MODEL_NAME` | 模型名称 | `claude-sonnet-4-20250514` |
