# TestPilot 测试领航 产品介绍文档

> 版本：v2.5
> 文档更新：2026-03-22

## 一、产品概述

**TestPilot（测试领航）** 是一款 AI 赋能的全栈接口测试管理平台。它将大语言模型（LLM）的推理能力与专业的测试工程实践深度融合，提供从测试用例生成、执行到结果管理的完整闭环。

用户通过自然语言聊天界面描述测试需求，AI Agent 自动生成测试用例、执行接口测试、输出 HTML 报告；测试结果自动写入用例库；基线用例可用于 AI 回归推荐，驱动下一轮回归测试。

**产品口号**：从对话到知识库，全程智能驾驶

## 六、四种测试模式

| 模式 | 核心特征 | 适用场景 |
|------|---------|---------|
| **systematic**（系统化） | BVA + 等价类驱动，用例 ID 顺序编号，禁止随机值 | 新业务规则首次建设用例库 |
| **regression**（回归） | 运行已有基线 YAML 用例，100% 确定性，CI/CD 友好 | 版本发布前回归验证 |
| **exploratory**（探索） | LLM 假设驱动，逐步分析、挖掘非常规边界缺陷 | 对抗性测试，发现未知缺陷 |
| **chaos**（混沌/对比） | 同一用例集分别跑 :8000 和 :8001 两套系统，自动差异分析 | 重构验证，新旧系统对比 |

## 七、主要功能模块

### 7.1 AI 对话与流式执行
- 自然语言描述测试需求，Agent 自动规划并执行
- Server-Sent Events（SSE）实时推送思考过程、工具调用、test_progress
- 支持中断正在运行的 Agent 任务
- 支持文件上传，Agent 可结合附件内容进行分析

### 7.2 测试用例库
- 用例保存为 DRAFT 后，经 submit→review→baseline 流程晋级
- TestCaseGroup 记录用例的逻辑身份（modelId + caseCode 唯一），每次更新生成新 TestCase 版本
- 版本历史完整可查，已基线用例不可修改，需新建版本
- 统计接口提供各状态用例数量汇总

### 7.3 AI 回归推荐
- 输入变更描述 → LLM 语义推理（结合用例关联的报告、对话、文档上下文）
- 推荐结果包含相关度分数（0-100）和推荐理由，支持追溯
- LLM 不可达时降级返回全部基线用例

### 7.4 文件下载与报告库
- Agent 输出 2 个 HTML 文件（测试用例 + 测试报告）通过下载面板提供
- 下载面板含"保存到报告库"按钮，触发报告入库流程
- 测试报告页：HTML 在线预览、查看已关联用例、确认用例入库

### 7.5 缺陷管理
- 手动创建缺陷（标题、严重级别 P0-P3、描述、关联报告/用例）
- 缺陷状态流转：待处理 → 处理中 → 已解决 → 已关闭 / 不修复
- 缺陷详情页：评论区、属性面板（状态更新、关联链接）

### 7.6 知识库管理（v2.5 新增）
- **业务规则库**：将业务规范文档（Markdown）持久化存储，对话时自动注入 Agent 上下文，避免 LLM 对业务规则的错误推断
- **挡板场景库**：为外部依赖（三方接口、数据库挡板等）定义命名场景，测试用例通过 `external_setup_ref` 引用，可复用、易维护
- **测试规范库**：团队共识的测试策略与规范（等价类划分、边界值要求等），注入系统提示词，统一 Agent 出用例的行为基准
- 三类知识库均支持按 modelId（接口模型）过滤，实现跨团队知识共享与复用

## 九、快速开始

### 环境要求
- Node.js 22+（路径：`/usr/local/Cellar/node/25.8.1_1/bin/`）
- PostgreSQL 16（`:5432`，DB：`testagent`）
- Redis 6+（`:6379`）

### 本地开发启动

```bash
# 后端（根目录，端口 8000）
PATH="/usr/local/Cellar/node/25.8.1_1/bin:$PATH" npm run dev

# 前端（frontend/ 目录，端口 3000）
cd frontend && PATH="/usr/local/Cellar/node/25.8.1_1/bin:$PATH" npm run dev
```

### 数据库初始化

```bash
npm run prisma:push      # 同步 schema（开发）
npm run prisma:migrate   # 创建并运行迁移
npm run prisma:generate  # 重新生成 Prisma Client
```

### 关键环境变量

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| `PORT` | 后端监听端口 | `8000` |
| `DATABASE_URL` | PostgreSQL 连接串 | `postgresql://mcp_user:mcp_password@localhost:5432/testagent?schema=public` |
| `REDIS_URL` | Redis 连接地址 | `redis://localhost:6379` |
| `JWT_SECRET` | JWT 签名密钥 | `<随机字符串>` |
| `LLM_PROVIDER` | LLM 提供商 | `openai` |

| `LLM_API_KEY` | API Key | `<API Key>` |

| `WORKSPACE_DIR` | Agent 工作区根目录 | `.testagent/workspace` |
| `AGENT_OUTPUT_DIR` | Agent 输出文件目录 | `.testagent/workspace/outputs` |

## 十、适用场景

- **测试工程师**：通过自然语言描述测试目标，AI 自动生成用例、执行接口测试、输出报告，将测试准备时间从数天压缩到分钟级
- **审核人 / 技术负责人**：管理用例库生命周期，晋级高质量用例为基线，一键触发 AI 回归推荐
- **DevOps 团队**：回归模式（regression）固化用例直接集成到 CI/CD 流程，实现自动化回归
