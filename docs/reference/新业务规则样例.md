授信额度模型 001 系统设计详细文档
1. 模型基本信息
•模型编号：MODEL_001
•模型名称：授信额度模型
•用途：根据用户社保、工资、收入、外部接口指标等信息，计算是否准入、是否产生授信额度
•授信额度模型 的所有输入指标来源：
1）数据库指标（从本地表读取）
2）外部接口指标（调用第三方系统，通过报文获取）
所有指标用于：准入判断 + 授信额度计算。
•输出：
￮准入/不准入
￮授信额度（不准入=0）
2. 所有数据表 + 字段 + 含义（完整版）
表 1：user_info（用户基础信息表）
•作用：用户基本信息
•关键字段：
￮user_level（int）：用户等级，取值1~5
表 2：account_balance（账户资产表）
•作用：用户资产能力
•关键字段：
￮avg_3m_balance（decimal）：近3个月月均资产，用户可为负资产
表 3：cgs_social_security（社保信息表）
•作用：记录用户是否缴纳社保
•关键字段：
￮social_security_flag（int，枚举）：取值0/1；0=无社保，1=有社保
表 4：salary_summary（工资汇总信息表）
•作用：记录用户近一个月工资收入（按月存储）
•关键字段：
￮monthly_salary（decimal）：近1个月工资总收入，范围1～10000000，无正负，仅为收入金额
3. 模型最终输入特征（共10个字段）
3.1 数据库表提取指标（6个）
1.user_info.user_level
2.account_balance.avg_3m_balance
3.cgs_social_security.social_security_flag（0/1）
4.salary_summary.monthly_salary
5.计算系数：coefficient
3.2 外部接口提取指标（4个）
1.card_status（借记卡状态）
2.recent_trans_amount（近30天借记卡交易金额）
3.id_check_result（身份核验结果）
4.is_black（是否在黑名单）
4. 业务规则
4.1 准入判断规则
必须同时满足以下6条，才算「准入」，任意一条不满足则「不准入」，额度直接为0：
1.近一个月工资 > 10000 元：monthly_salary > 10000
2.有社保：social_security_flag = 1
3.借记卡状态正常：card_status = "NORMAL"
4.身份核验通过：id_check_result = "PASS"
5.未进入黑名单：is_black = false
6.近30天借记卡有交易：recent_trans_amount > 0
4.2 计算系数取值规则
仅满足准入条件后才计算系数，不满足准入条件则计算系数取0：
1.准入且近3个月月均资产avg_3m_balance > 1000 → 计算系数取2.3
2.准入且近3个月月均资产0 < avg_3m_balance ≤ 1000 → 计算系数取0.5
3.准入且近3个月月均资产avg_3m_balance ≤ 0 → 计算系数取0.2
4.不满足准入条件 → 计算系数取0
4.3 授信额度计算公式（仅准入时计算）
满足准入条件后，按以下公式计算授信额度，若计算结果为负数，返回额度为0：
额度crgln =（user_level × avg_3m_balance × monthly_salary）× 计算系数

6.3 外部接口指标交易及返回报文
模型需调用第三方外部接口获取核验指标，用于准入判断，所有接口请求/响应均为XML格式，具体接口及提取指标如下：
6.3.1：借记卡交易信息查询-A00110001
•接口用途：查询用户借记卡是否正常、近段时间是否有交易
•请求报文：
XML
<?xml version="1.0" encoding="UTF-8"?>
<request>
  <user_id>字符串</user_id>
  <id_card>身份证号</id_card>
  <query_type>BALANCE</query_type>
</request>
•响应报文：
XML
<?xml version="1.0" encoding="UTF-8"?>
<response>
  <code>0000</code>
  <message>成功</message>
  <data>
    <card_no>6222****1234</card_no>
    <card_status>NORMAL</card_status>
    <recent_trans_amount>5800.50</recent_trans_amount>
    <trans_count>12</trans_count>
  </data>
</response>
•提取外部指标：
a.card_status（枚举）：借记卡状态，NORMAL=正常，ABNORMAL=异常
b.recent_trans_amount（decimal）：近30天借记卡交易金额
6.3.2：身份证信息核验-A00110002
•接口用途：验证身份证是否真实、是否为本人
•请求报文：
XML
<?xml version="1.0" encoding="UTF-8"?>
<request>
  <user_id>字符串</user_id>
  <id_card>身份证号</id_card>
  <user_name>姓名</user_name>
</request>
•响应报文：
XML
<?xml version="1.0" encoding="UTF-8"?>
<response>
  <code>0000</code>
  <message>成功</message>
  <data>
    <id_check_result>PASS</id_check_result>
    <id_valid>true</id_valid>
  </data>
</response>
•提取外部指标：
￮id_check_result（枚举）：身份核验结果，PASS=通过，FAIL=不通过
6.3.3：黑名单校验接口-A00110003
•接口用途：检查用户是否在行业/机构黑名单
