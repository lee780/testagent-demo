## 2. 测试流程图

```mermaid
sequenceDiagram
    participant A as 🤖 测试 Agent
    participant C as 授信 C 系统
    participant DB as 🗄️ 本地数据库
    participant S as 🧱 挡板系统

    Note over A,S: 阶段一：准备测试前置条件

    A->>DB: ② 写入本地指标<br/>（月薪、余额、社保等）
    A->>S: ③ 配置挡板返回值<br/>（支付宝特征、公积金缴存等）

    Note over A,S: 阶段二：发起测试请求

    A->>C: ① POST /credit/score { userId }
    activate C

    C->>DB: ④ 查本地沉淀指标
    DB-->>C: 返回历史流水、余额、社保状态

    C->>S: ⑤ 外呼支付宝（挡板拦截）
    S-->>C: 返回配置好的支付宝特征

    C->>S: ⑤ 外呼微信（挡板拦截）
    S-->>C: 返回配置好的微信流水

    C->>S: ⑤ 外呼公积金（挡板拦截）
    S-->>C: 返回配置好的公积金数据

    C->>C: 汇总本地指标 + 外呼指标<br/>按规则计算 → 准入 + 额度

    C-->>A: 返回 { admit_flag, credit_limit }
    deactivate C

    Note over A,S: 阶段三：结果比对

    A->>A: 实际结果 vs 预期结果<br/>判断用例通过 / 失败
```
