## 二、Mock Service 详细设计

### 2.1 `src/server/modules/mock/mock.service.ts`

**函数签名：**
```typescript
export async function calculateCreditScore(userId: string): Promise<ScoreResult>
```

**业务逻辑流程图：**
```
输入 userId
     │
     ▼
并发查询4张表
     │
     ├── 任一表无记录 → result_code=0001
     │
     ▼
准入判断：
  monthly_salary > 10000 AND social_security_flag = 1
     │
     ├── 不满足 → admit_flag=0, credit_limit=0.00
     │
     ▼ 满足（准入）
系数取值：
  avg_3m_balance > 0 → coefficient = 2.3
  avg_3m_balance ≤ 0 → coefficient = 0.5
     │
     ▼
额度计算：
  crgln = user_level × (avg_3m_balance/1000) × monthly_salary × coefficient
  crgln < 0 → crgln = 0
     │
     ▼
返回 admit_flag=1, credit_limit=crgln.toFixed(2)
```

**关键注意点：**
- `avg_3m_balance` 存储单位为元，公式中除以 1000 转换为千元单位
- `toFixed(2)` 保证两位小数格式
- 四张表并发查询（`Promise.all`）提升性能

}

## 三、Agent System Prompt 详细设计

### 3.1 修改文件：`src/agent-core/system-prompt.ts`

在现有 Prompt 基础上新增 **测试用例生成能力** 章节：

```
## Test Case Generation (接口测试用例生成)

当用户要求根据业务文档生成测试用例时，遵循以下流程：

### Step 1: 文档解析
1. 读取上传的业务规范文档
2. 提取：数据表结构、字段含义、准入规则、计算公式
3. 识别所有决策因子及其取值范围

### Step 2: 因子组合枚举
规则：
- 对影响"是否准入"的因子：穷举所有组合
- 对影响"额度计算"的因子（只在准入时有效）：
  * 连续值因子（如 avg_3m_balance）：取边界值（<0, =0, >0）
  * 离散值因子（如 user_level 1~5）：全枚举
- 合并冗余：不准入场景下，只保留代表性值

### Step 3: 预期值计算
- 严格按业务规则公式计算，不依赖估算
- credit_limit 保留两位小数（toFixed(2)）
- 明确标注使用的系数值

### Step 4: 输出 cases_summary.json
- 用 write_output_file 写入 cases_summary.json
- 格式严格按照规范（含 id/name/category/preconditions/expected/business_rules）

### Step 5: 生成 SQL + YAML（人工确认后）
SQL 规范：
- DELETE 先于 INSERT（幂等）
- 每张数据表各一条 DELETE 和 INSERT
- 文件名：TC_MODEL001_{NNN}_setup.sql

YAML 规范（HTTPRunner v4）：
- config.base_url 使用 ${ENV(BASE_URL)}
- 断言使用 contains 匹配 XML 片段
- 文件名：TC_MODEL001_{NNN}_test.yaml
```

## 四、Vitest 单元测试运行设计

### 4.1 测试辅助函数 `test/mock/helpers.ts`

```typescript
import { getPrisma } from '../../src/server/config/database.js'

interface UserSetup {
  userLevel: number
  balance: number
  ss: number
  salary: number
}

export async function setupUser(userId: string, data: UserSetup) {
  const db = getPrisma()
  await db.$transaction([
    db.mockUserInfo.upsert({
      where: { userId }, create: { userId, userLevel: data.userLevel },
      update: { userLevel: data.userLevel }
    }),
    db.mockAccountBalance.upsert({
      where: { userId }, create: { userId, avg3mBalance: data.balance },
      update: { avg3mBalance: data.balance }
    }),
    db.mockSocialSecurity.upsert({
      where: { userId }, create: { userId, socialSecurityFlag: data.ss },
      update: { socialSecurityFlag: data.ss }
    }),
    db.mockSalarySummary.upsert({
      where: { userId }, create: { userId, monthlySalary: data.salary },
      update: { monthlySalary: data.salary }
    }),
  ])
}

export async function cleanupUser(userId: string) {
  const db = getPrisma()
  await db.$transaction([
    db.mockUserInfo.deleteMany({ where: { userId } }),
    db.mockAccountBalance.deleteMany({ where: { userId } }),
    db.mockSocialSecurity.deleteMany({ where: { userId } }),
    db.mockSalarySummary.deleteMany({ where: { userId } }),
  ])
}
```

## 六、关键算法：因子组合枚举

**伪代码（Agent 推理逻辑）：**

```
输入因子：
  F1 = monthly_salary: [<=10000, >10000]
  F2 = social_security_flag: [0, 1]
  F3 = avg_3m_balance: [<0, =0, >0]（仅准入时有意义）
  F4 = user_level: [1, 2, 3, 4, 5]（仅准入且avg≠0时有意义）

枚举逻辑：
  for F1, F2 in cross_product([F1_values, F2_values]):
    admitted = (F1 == '>10000') and (F2 == 1)
    if not admitted:
      emit case(F1, F2, avg=0, level=1)  # 代表值即可
    else:
      for F3 in [<0, =0, >0]:
        if F3 == '<0' or F3 == '=0':
          emit case(F1=12000, F2=1, F3, level=1)  # 额度归零，level无影响
        else:  # F3 > 0
          for F4 in [1,2,3,4,5]:
            emit case(F1=15000, F2=1, F3=5000, F4)
```

**预期值计算：**
```
if not admitted:
  admit_flag = '0', credit_limit = '0.00', coefficient = 0
else:
  coefficient = 2.3 if avg_3m_balance > 0 else 0.5
  crgln = user_level * (avg_3m_balance / 1000) * monthly_salary * coefficient
  crgln = max(crgln, 0)
  admit_flag = '1', credit_limit = f'{crgln:.2f}'
```
