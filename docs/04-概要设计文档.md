## 三、完整数据流

```
Step 1: 文档解析 → JSON 用例

用户上传文档 ──→ chat.routes.ts (文件保存)
                        │
                        ▼
          Agent runner.ts (读文档, 分析规则)
                        │
                        ▼
          LLM 推理 (因子枚举 + 预期值计算)
                        │
                        ▼
          write_output_file → cases_summary.json
                        │
                        ▼
          SSE → 前端展示 + 用户下载 Review


Step 2: JSON → SQL + YAML

用户确认指令 ──→ Agent runner.ts
                        │
                 读取 cases_summary.json
                        │
                  for each case:
                  ├── write SQL → sql/TC_XXX_setup.sql
                  └── write YAML → testcases/TC_XXX_test.yaml
                        │
                        ▼
                  SSE → 前端展示文件列表


Step 3: 执行 → 报告

用户执行指令 ──→ Agent runner.ts
                        │
                  bash: 执行所有 *.sql
                        │
                  bash: httprunner run testcases/
                        │
                  读取执行结果 / HTML 报告路径
                        │
                  write_output_file → summary.md
                        │
                        ▼
                  SSE → Markdown 摘要输出
```

## 四、接口规格

### 4.1 被测接口（已实现）

```
POST /mock/model/score
Content-Type: application/xml

Request:
  <REQUEST>
    <user_id>{string}</user_id>
    <model_id>MODEL_001</model_id>
    <channel>131</channel>
  </REQUEST>

Response:
  <RESPONSE>
    <result_code>{0000|0001|9999}</result_code>
    <result_msg>{string}</result_msg>
    <admit_flag>{0|1}</admit_flag>
    <credit_limit>{decimal:.2f}</credit_limit>
  </RESPONSE>
```

### 4.2 文件输出规范

**cases_summary.json**：
```json
[
  {
    "id": "TC_MODEL001_NNN",
    "name": "{string}",
    "category": "正常流程|边界值|异常流程",
    "preconditions": {
      "user_level": {int},
      "avg_3m_balance": {decimal},
      "social_security_flag": {0|1},
      "monthly_salary": {decimal}
    },
    "expected": {
      "admit_flag": "{0|1}",
      "credit_limit": "{decimal:.2f}",
      "coefficient": {0|0.5|2.3}
    },
    "business_rules": ["{string}"]
  }
]
```

**TC_XXX_setup.sql**：
```sql
-- 注释：用例ID、名称
DELETE FROM mock_user_info WHERE user_id = '{id}';
DELETE FROM mock_account_balance WHERE user_id = '{id}';
DELETE FROM mock_social_security WHERE user_id = '{id}';
DELETE FROM mock_salary_summary WHERE user_id = '{id}';
INSERT INTO mock_user_info VALUES ('{id}', {user_level});
INSERT INTO mock_account_balance VALUES ('{id}', {avg_3m_balance});
INSERT INTO mock_social_security VALUES ('{id}', {social_security_flag});
INSERT INTO mock_salary_summary VALUES ('{id}', {monthly_salary});
```

**TC_XXX_test.yaml**（HTTPRunner v4）：
```yaml
config:
  name: "{TC_ID} {name}"
  base_url: "${ENV(BASE_URL)}"

teststeps:
  - name: "调用授信测算接口"
    request:
      url: /mock/model/score
      method: POST
      headers:
        Content-Type: application/xml;charset=utf-8
      body: |
        <?xml version="1.0" encoding="utf-8"?>
        <REQUEST>
          <user_id>{id}</user_id>
          ...
        </REQUEST>
    validate:
      - eq: ["status_code", 200]
      - contains: ["body", "<result_code>0000</result_code>"]
      - contains: ["body", "<admit_flag>{0|1}</admit_flag>"]
      - contains: ["body", "<credit_limit>{decimal}</credit_limit>"]
```

## 六、错误处理策略

| 场景 | 处理方式 |
|------|---------|
| 文档格式不符合预期 | Agent 输出错误说明，要求用户确认 |
| 预期值计算失败 | 标记该用例为"待人工确认"，不阻塞其他用例 |
| SQL 执行报错 | 输出报错信息，停止后续执行 |
| httprunner 未安装 | 给出安装命令提示，不抛出系统错误 |
| HTTPRunner 执行失败 | 输出 stderr，保留已有报告 |
