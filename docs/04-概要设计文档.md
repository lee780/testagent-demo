## 二、模块划分

### 2.1 后端模块

| 模块 | 路径 | 职责 |
|------|------|------|
| 认证模块 | `src/server/modules/auth/` | 注册/登录/注销/JWT/Session/密码修改 |
| 对话模块 | `src/server/modules/conversation/` | 对话和消息 CRUD |
| 聊天模块 | `src/server/modules/chat/` | SSE 流式对话、文件上传下载、Agent 中断 |
| 测试用例模块 | `src/server/modules/testcase/` | 用例 CRUD、生命周期、版本控制、AI 推荐 |
| 测试报告模块 | `src/server/modules/report/` | 报告 CRUD、HTML 预览、用例确认入库 |
| 缺陷模块 | `src/server/modules/defect/` | 缺陷 CRUD、状态流转、评论 |
| Mock 模块 | `src/server/modules/mock/` | MODEL_001 授信额度测算被测接口 |

| 模块 | 路径 | 职责 |
|------|------|------|

## 三、数据库设计

### 3.1 核心业务表

| 表名 | 说明 | 关键字段 |
|------|------|---------|
| `users` | 用户 | id, username, role(USER/ADMIN), status(ACTIVE/INACTIVE/BANNED) |
| `conversations` | 对话会话 | id, userId, title |
| `messages` | 消息 | id, conversationId, role, content, metadata（含 SSE 事件回放） |
| `agent_sessions` | Agent 执行记录 | id, conversationId, replyId, status, agentType |
| `coordinator_plans` | 多任务协调计划 | conversationId(unique), plan(JSON), activePhase |

### 3.2 测试用例库表

| 表名 | 说明 | 关键字段 |
|------|------|---------|
| `test_case_groups` | 用例逻辑身份 | id, modelId, caseCode, latestId（unique: modelId+caseCode） |
| `test_cases` | 用例版本快照 | id, groupId, version, status, title, inputParams, expectedResult, yamlContent, sourceMode |
| `test_executions` | 执行记录 | id, testCaseId, conversationId, status(PASS/FAIL/ERROR/SKIP), durationMs |

TestCase 状态枚举：`DRAFT` → `PENDING_REVIEW` → `APPROVED` → `BASELINE` → `DEPRECATED`

### 3.4 测试报告与缺陷表

| 表名 | 说明 | 关键字段 |
|------|------|---------|
| `test_reports` | 测试执行报告 | id, name, conversationId, executionMode, htmlFile, testCasesData(JSON), casesImported |
| `defects` | 缺陷记录 | id, title, status(待处理/处理中/已解决/已关闭/不修复), severity(P0-P3), reportId, testCaseId |
| `defect_comments` | 缺陷评论 | id, defectId, content, createdBy |

### 3.3 Mock 被测系统表

| 表名 | 说明 |
|------|------|
| `mock_user_info` | 用户等级（userId, userLevel） |
| `mock_account_balance` | 三月平均余额（userId, avg3mBalance） |
| `mock_social_security` | 社保标志（userId, socialSecurityFlag） |
| `mock_salary_summary` | 月薪（userId, monthlySalary） |

## 五、mode 参数数据流

```
前端 currentMode ref（默认 'systematic'）
    ↓ fetch body: { message, mode: currentMode.value, conversationId }
chat.schemas.ts: z.enum(['regression','systematic','exploratory','chaos']).optional()
    ↓ 非法值 → HTTP 400 VALIDATION_ERROR
chat.routes.ts: body.mode → sendMessageStreaming({ mode })
    ↓
chat.service.ts: mode → runTestAgent({ mode })
    ↓
runner.ts: mode → buildSystemPrompt({ mode })
    ↓
system-prompt.ts: switch(mode) → 4 套独立 System Prompt
```

## 六、测试报告与用例入库数据流

```
Agent 执行测试（test-runner.ts）
    ↓ 测试完成，调用 saveRawTestCases()
.testagent/workspace/outputs/{userId}/{convId}/test_cases_raw.json
    ↓ 用户点击下载面板"保存到报告库"按钮
POST /api/reports → report.service.ts createReport()
    ↓ 读取 test_cases_raw.json 存入 TestReport.testCasesData
数据库写入 test_reports 表（casesImported=false）
    ↓ 用户在报告列表页点击"确认入库"
POST /api/reports/:id/import → report.service.ts importTestCases()
    ↓ 读取 TestReport.testCasesData，逐条创建 TestCase（status=DRAFT）
    ↓ 更新 TestReport.casesImported = true
前端用例库页面可查到新 DRAFT 用例
```

## 七、工作区隔离

每个对话的输出文件路径：
```
{AGENT_OUTPUT_DIR}/{userId}/{conversationId}/
├── {ModelId}_测试用例.html    # 测试用例 HTML
├── {ModelId}_测试报告.html    # 测试报告 HTML
└── test_cases_raw.json        # 原始用例数据（用于手动入库）
```

下载接口：`GET /api/conversations/:id/outputs/:filename`
- 服务端返回 `Content-Type: application/octet-stream`
- 前端预览 HTML 需用 `new Blob([html], { type: 'text/html' })` 重建 MIME 类型再 `window.open`

## 八、技术选型

| 组件 | 选型 | 版本 | 理由 |
|------|------|------|------|
| 后端框架 | Fastify | 5.x | 高性能，TypeScript 友好 |
| ORM | Prisma | 6.x | 类型安全，迁移管理方便 |
| AI 框架 | pi-mono SDK | 0.55.x | 支持多 LLM Provider，内置 Agent 工具体系 |
| 测试框架 | Vitest | 3.x | 与 TypeScript 生态无缝集成 |
| 前端框架 | Vue 3.5 + Vite | 最新 | Composition API，HMR 快速迭代 |
| 数据库 | PostgreSQL | 16 | 关系型，支持 JSONB，事务完整 |
| 缓存 | Redis | 6+ | Session 存储，中断信号传递 |
| 入参校验 | Zod | 3.x | 运行时类型安全，与 TypeScript 推导联动 |

## 九、错误处理策略

| 场景 | 处理方式 |
|------|---------|
| 非法 mode 值 | Zod 枚举校验，HTTP 400 VALIDATION_ERROR |
| 用例状态流转违规 | service 层抛出 Error，routes 层捕获返回 HTTP 400 |
| 缺陷不存在 | routes 层 HTTP 404 |
| Agent 执行失败 | SSE error 事件推送，AgentSession.status = FAILED |
| 文件不存在 | 下载接口 HTTP 404 |
| JWT 失效 | 认证插件 HTTP 401 |
| 已知 Bug：注销后 Token 仍有效 | 待修复（auth.ts:59 sessionData null 时需拒绝而非放行） |
